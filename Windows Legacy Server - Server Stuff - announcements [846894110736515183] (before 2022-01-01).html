Ximonite 23/08/2021 08:22
Vista on Haswell and newer - the full explanation: My theory is that there were originally two big issues with the Vista HAL: Corruption and drift. The corruption issue does exactly what it sounds like it does. The system starts corrupting itself. This issue was fixed in the Build 6003 HAL, but the drift issue still remains, explaining the remaining issues, including services not starting properly and various DirectX 11 issues. Early in the Windows 7 development process, drift correction was added to the HAL. The reason is unknown, but this happens to be needed for NT 6 based OSes to run optimally on Haswell and newer Intel CPUs. The drift originates from Intel switching the clock circuit used in their CPUs during Haswell development. This new clock circuit was easier and cheaper to produce but also less precise. (edited)
docR 28/08/2021 06:56
To add to the aforementioned. Disabling hyperthreading, if applicable, also seems to reduce the frequency of disrupted startups. Furthermore you should remove intelppm and set the CPU core drivers to default/generic via Device Manager. DirectX 11 issues may be (partially) mitigated as well by using nvwgf2umx.dll from GeForce driver version 285 through 335 (285 works best as newer versions may throw an error). This is sort of a pseudo-fix, as DirectX feature level appears to be at 10.3. Programs that use DirectX 11 in some limited capacity should now work though. webgl no longer crashes Firefox-based browsers, Discord streaming from Chrome now works, etc. (edited)
